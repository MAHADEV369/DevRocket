# Skill 20: PWA Service Worker for Web Frontend

Version: 1.0.0
Last Updated: 2025-01-15
Estimated Time: 1-2 hours
Depends On: 23-frontend-setup

## Input Contract
- Vite-based frontend project (Skill 23) with `vite.config.ts`
- React (or similar framework) SPA in the `web/` directory
- `package.json` at `web/package.json`
- API endpoints at `/api/v1/` served by the backend
- Static assets (icons, images) available or placeholderable

## Output Contract
- PWA manifest with app name, icons, theme colors, and display mode
- Service worker auto-generated by Workbox via `vite-plugin-pwa`
- Runtime caching strategies: NetworkFirst for API calls, CacheFirst for images
- Auto-update behavior prompting users on new version
- Installable PWA verified via Lighthouse audit

## Files to Create
- `web/vite.config.ts` — Updated with PWA plugin configuration (or modifications to existing)
- `web/public/icons/` — PWA icon assets (placeholder if needed)
- `web/src/sw-register.ts` — Service worker registration (if not auto-injected)

### Step 3: Configure `vite-plugin-pwa` in `web/vite.config.ts`

If `vite.config.ts` already exists, update it. If not, create it:

```typescript
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';
import { VitePWA } from 'vite-plugin-pwa';

export default defineConfig({
  plugins: [
    react(),
    VitePWA({
      registerType: 'autoUpdate',
      includeAssets: [
        'icons/favicon-32x32.png',
        'icons/favicon-16x16.png',
        'icons/apple-touch-icon.png',
      ],
      manifest: {
        name: 'Image App',
        short_name: 'ImageApp',
        description: 'Upload, process, and generate AI-powered images',
        theme_color: '#2563EB',
        background_color: '#ffffff',
        display: 'standalone',
        orientation: 'portrait-primary',
        scope: '/',
        start_url: '/',
        icons: [
          {
            src: 'icons/icon-192x192.png',
            sizes: '192x192',
            type: 'image/png',
          },
          {
            src: 'icons/icon-512x512.png',
            sizes: '512x512',
            type: 'image/png',
          },
          {
            src: 'icons/icon-512x512.png',
            sizes: '512x512',
            type: 'image/png',
            purpose: 'maskable',
          },
        ],
      },
      workbox: {
        // Runtime caching strategies
        runtimeCaching: [
          // API calls — NetworkFirst (fresh data, fall back to cache)
          {
            urlPattern: /^https?:\/\/.*\/api\/v1\/.*/i,
            handler: 'NetworkFirst',
            options: {
              cacheName: 'api-cache',
              expiration: {
                maxEntries: 100,
                maxAgeSeconds: 60 * 5, // 5 minutes
              },
              networkTimeoutSeconds: 10,
              cacheableResponse: {
                statuses: [0, 200],
              },
              headers: {
                'Cache-Control': 'no-cache',
              },
            },
          },

          // Auth endpoints — NetworkOnly (never cache auth responses)
          {
            urlPattern: /^https?:\/\/.*\/api\/v1\/auth\/.*/i,
            handler: 'NetworkOnly',
          },

          // Image uploads — NetworkOnly (mutations should always hit network)
          {
            urlPattern: /^https?:\/\/.*\/api\/v1\/images\/(upload|generate)/i,
            handler: 'NetworkOnly',
          },

          // Static images — CacheFirst (cache for performance)
          {
            urlPattern: /\.(?:png|jpg|jpeg|svg|gif|webp)$/i,
            handler: 'CacheFirst',
            options: {
              cacheName: 'image-cache',
              expiration: {
                maxEntries: 200,
                maxAgeSeconds: 60 * 60 * 24 * 30, // 30 days
              },
              cacheableResponse: {
                statuses: [0, 200],
              },
            },
          },

          // Fonts — CacheFirst (rarely change)
          {
            urlPattern: /\.(?:woff2?|ttf|otf|eot)$/i,
            handler: 'CacheFirst',
            options: {
              cacheName: 'font-cache',
              expiration: {
                maxEntries: 20,
                maxAgeSeconds: 60 * 60 * 24 * 365, // 1 year
              },
              cacheableResponse: {
                statuses: [0, 200],
              },
            },
          },

          // Static assets (JS, CSS) — StaleWhileRevalidate
          {
            urlPattern: /\.(?:js|css)$/i,
            handler: 'StaleWhileRevalidate',
            options: {
              cacheName: 'static-assets',
              expiration: {
                maxEntries: 100,
                maxAgeSeconds: 60 * 60 * 24 * 7, // 7 days
              },
              cacheableResponse: {
                statuses: [0, 200],
              },
            },
          },
        ],
        // Precache static assets built by Vite
        globPatterns: ['**/*.{js,css,html,ico,png,svg,woff2}'],
        // Skip waiting and claim clients on new version
        skipWaiting: true,
        clientsClaim: true,
      },
      devOptions: {
        enabled: false, // Disable SW in dev for hot-reload compatibility
      },
    }),
  ],
  server: {
    proxy: {
      '/api': {
        target: 'http://localhost:3000',
        changeOrigin: true,
      },
    },
  },
});
```

### Step 4: Add Service Worker Registration to App Entry Point

If `vite-plugin-pwa` with `registerType: 'autoUpdate'` does not auto-register, add explicit registration. Create `web/src/sw-register.ts`:

```typescript
if ('serviceWorker' in navigator) {
  window.addEventListener('load', () => {
    navigator.serviceWorker
      .register('/sw.js')
      .then((registration) => {
        console.log('SW registered:', registration.scope);

        registration.addEventListener('updatefound', () => {
          const newWorker = registration.installing;
          if (!newWorker) return;

          newWorker.addEventListener('statechange', () => {
            if (newWorker.state === 'activated') {
              console.log('New service worker activated — refresh for updates');
            }
          });
        });
      })
      .catch((error) => {
        console.error('SW registration failed:', error);
      });
  });
}
```

Import it in your main entry file (`web/src/main.tsx` or `web/src/main.ts`):

```typescript
import './sw-register';
```

### Step 6: Add Update Notification Component

Create `web/src/components/UpdateNotification.tsx`:

```tsx
import { useState, useEffect } from 'react';

interface BeforeInstallPromptEvent extends Event {
  prompt(): Promise<void>;
  userChoice: Promise<{ outcome: 'accepted' | 'dismissed' }>;
}

export function UpdateNotification() {
  const [updateAvailable, setUpdateAvailable] = useState(false);
  const [installPrompt, setInstallPrompt] = useState<BeforeInstallPromptEvent | null>(null);
  const [showInstallBanner, setShowInstallBanner] = useState(false);

  useEffect(() => {
    // Capture the beforeinstallprompt event for custom install UI
    const handler = (e: Event) => {
      e.preventDefault();
      setInstallPrompt(e as BeforeInstallPromptEvent);
      setShowInstallBanner(true);
    };
    window.addEventListener('beforeinstallprompt', handler);

    // Listen for service worker updates
    if ('serviceWorker' in navigator) {
      navigator.serviceWorker.addEventListener('controllerchange', () => {
        setUpdateAvailable(true);
      });
    }

    return () => {
      window.removeEventListener('beforeinstallprompt', handler);
    };
  }, []);

  const handleUpdate = () => {
    window.location.reload();
  };

  const handleInstall = async () => {
    if (!installPrompt) return;
    await installPrompt.prompt();
    const { outcome } = await installPrompt.userChoice;
    if (outcome === 'accepted') {
      setShowInstallBanner(false);
    }
    setInstallPrompt(null);
  };

  return (
    <>
      {updateAvailable && (
        <div style={{
          position: 'fixed',
          bottom: 0,
          left: 0,
          right: 0,
          padding: '12px 16px',
          background: '#2563EB',
          color: 'white',
          textAlign: 'center',
          zIndex: 9999,
        }}>
          A new version is available.
          <button
            onClick={handleUpdate}
            style={{
              marginLeft: '12px',
              padding: '6px 16px',
              background: 'white',
              color: '#2563EB',
              border: 'none',
              borderRadius: '4px',
              cursor: 'pointer',
              fontWeight: 'bold',
            }}
          >
            Update Now
          </button>
        </div>
      )}

      {showInstallBanner && (
        <div style={{
          position: 'fixed',
          bottom: updateAvailable ? '60px' : 0,
          left: 0,
          right: 0,
          padding: '12px 16px',
          background: '#1E293B',
          color: 'white',
          textAlign: 'center',
          zIndex: 9998,
        }}>
          Install Image App for a better experience.
          <button
            onClick={handleInstall}
            style={{
              marginLeft: '12px',
              padding: '6px 16px',
              background: '#2563EB',
              color: 'white',
              border: 'none',
              borderRadius: '4px',
              cursor: 'pointer',
              fontWeight: 'bold',
            }}
          >
            Install
          </button>
          <button
            onClick={() => setShowInstallBanner(false)}
            style={{
              marginLeft: '8px',
              padding: '6px 16px',
              background: 'transparent',
              color: '#94A3B8',
              border: '1px solid #475569',
              borderRadius: '4px',
              cursor: 'pointer',
            }}
          >
            Not Now
          </button>
        </div>
      )}
    </>
  );
}
```

Add it to your root component (e.g., `App.tsx`):

```tsx
import { UpdateNotification } from './components/UpdateNotification';

function App() {
  return (
    <>
      {/* ... existing app content ... */}
      <UpdateNotification />
    </>
  );
}
```

## Verification

1. **Build the project:**
   ```bash
   cd web && npm run build
   ```
   Expected: Build succeeds, `dist/manifest.webmanifest` and `dist/sw.js` are generated.

2. **Preview the production build:**
   ```bash
   npm run preview
   ```
   Expected: App serves on localhost with SW registered.

3. **Check manifest in browser:**
   - Open `http://localhost:4173/manifest.webmanifest`
   - Expected: Valid JSON with `name`, `short_name`, `icons`, `theme_color`, `display: standalone`

4. **Verify service worker registration:**
   - Open DevTools → Application → Service Workers
   - Expected: One SW registered with status "activated and is running"

5. **Verify caching strategies:**
   - Open DevTools → Application → Cache Storage
   - Expected: `api-cache`, `image-cache`, `font-cache`, `static-assets` and `workbox-precache` caches exist

6. **Run Lighthouse PWA audit:**
   - Open DevTools → Lighthouse → check "Progressive Web App"
   - Expected: PWA score ≥ 90%

7. **Test offline behavior:**
   - Load the app fully
   - In DevTools → Network → check "Offline"
   - Reload
   - Expected: App shell loads from cache, API calls show cached data or fail gracefully

8. **Test install prompt:**
   - In Chrome, look for the install icon in the address bar
   - Expected: "Install ImageApp" prompt appears

## Rollback

1. Uninstall PWA plugin:
   ```bash
   cd web && npm uninstall vite-plugin-pwa
   ```

2. Remove PWA config from `web/vite.config.ts`:
   - Remove the `VitePWA` plugin import and configuration
   - Keep the `react()` plugin and `server.proxy` config

3. Delete PWA-specific files:
   ```bash
   rm -rf web/public/icons
   rm web/src/sw-register.ts
   rm web/src/components/UpdateNotification.tsx
   ```

4. Remove PWA meta tags from `web/index.html` — keep only basic HTML.

5. Remove SW registration import from `web/src/main.tsx`.

6. Clean build output:
   ```bash
   rm -rf web/dist
   ```

## ADR-020: vite-plugin-pwa with Workbox Runtime Caching

**Decision:** Use `vite-plugin-pwa` with Workbox runtime caching strategies, auto-update registration, and custom manifest configuration.

**Reason:** `vite-plugin-pwa` integrates seamlessly with the Vite build pipeline and generates the service worker automatically during production builds. Workbox provides proven caching strategies out of the box. NetworkFirst for API calls ensures users see fresh data when online but still get cached responses during brief connectivity drops. CacheFirst for images and fonts reduces redundant network requests. Auth and mutation endpoints use NetworkOnly to prevent stale/cached sensitive data.

**Consequences:**
- Service worker is only generated during `npm run build`, not during development (configured via `devOptions.enabled: false`)
- Users receive automatic update notifications via the `UpdateNotification` component
- Cache invalidation is tied to the Vite build hash — new deployments automatically trigger SW updates
- NetworkFirst API caching may serve stale data briefly (up to 5 minutes cache age) on reconnect
- PWA install requires HTTPS in production — works on localhost for development

**Alternatives Considered:**
- **Manual service worker (`sw.js`):** Full control but high maintenance burden, easy to introduce bugs
- **Workbox InjectManifest:** More control than GenerateSW but requires writing SW logic manually
- **Next.js PWA:** Not applicable — we use Vite, not Next.js
- **No service worker:** Simplest but no offline capability, no installability, lower Lighthouse PWA score
